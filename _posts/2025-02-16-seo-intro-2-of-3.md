> 关键词提示：站内 SEO、站外 SEO、技术 SEO、搜索引擎优化、网站结构。

在《**SEO 入门必看 1/3**》中，我们聊了 SEO（搜索引擎优化）的概念与发展历史。这一篇，我们来系统梳理 SEO 的常见技术与做法。

很多人一提到 SEO，就只想到「写关键词」和「发外链」，但在现代搜索引擎下，一个完整的 SEO 策略通常包括三部分：

1. **站内 SEO（On-page SEO）**
2. **站外 SEO（Off-page SEO）**
3. **技术 SEO（Technical SEO）**

## 一、站内 SEO：让每一篇页面都容易被理解

站内 SEO 重点在于「**页面本身**」：标题、内容结构、关键词布局、内链等。目标是让搜索引擎理解：

> 这个页面在讲什么？适合什么搜索词？是否高质量？

### 1. 标题（Title）与标题标签（H1/H2/H3）

- 页面 `<title>` 和文章主标题（通常是 H1）应该包含 **核心关键词**。
- 小标题（H2/H3）可以自然使用相关词和长尾关键词。

**示例（以 SEO 入门为例）：**

- 标题：`SEO 入门必看：什么是搜索引擎优化？`  
- H2：`SEO 的基本概念`、`搜索引擎是如何排名的？`

### 2. 合理的关键词布局

现代 SEO 不鼓励关键词堆砌，但仍然建议：

- 在标题、首段、结尾自然出现核心关键词（如「SEO 入门」「搜索引擎优化」）；
- 在正文中用自然语言描述相关概念和长尾问题，比如「什么是技术 SEO」「如何优化网站结构」；
- 避免生硬重复同一个词。

### 3. 内链（Internal Links）

站内链接是 SEO 的重要部分：

- 在一篇文章中自然地链接到你站内的其他相关内容；
- 比如：在这篇讲 SEO 技术的文章里，链接回《SEO 入门必看 1/3》。

这样可以帮助：

- 搜索引擎爬虫更好地抓取和索引你的网站；
- 用户在站内浏览更多页面，提升网站整体价值。

### 4. 内容质量与可读性

良好的站内 SEO 不是堆词，而是：

- 内容结构清晰：有目录、有小标题、有段落；
- 用例子解释抽象概念（比如用一个简单的网站结构图说明 URL 组织方式）；
- 用代码块、列表等方式提高可读性（尤其是技术博客）。

## 二、站外 SEO：来自网站之外的信号

站外 SEO 的核心是「**别人在互联网上如何提到你**」。

典型的站外 SEO 信号包括：

1. **外链（Backlinks）**
   - 别的网站链接到你的网站。
   - 高质量的外链，会提高你在搜索引擎眼中的「权威度」。

2. **品牌与搜索量**
   - 用户主动搜你的品牌名（比如「某某博客 SEO 入门」）。
   - 对搜索引擎来说，这说明你有一定品牌价值。

3. **社交信号与口碑**（间接）
   - 虽然「社交分享」本身不是直接排名因子，但会带来更多自然外链和品牌曝光。

### 合理做站外 SEO 的建议

- 把时间更多花在 **产出高质量内容** 上，自然会有人引用和链接你的文章；
- 在相关社区（如技术论坛、GitHub README、开源项目文档）合理添加你网站的链接；
- 避免批量购买垃圾外链和不相关目录，这类做法在现代 SEO 中风险很高。

## 三、技术 SEO：让网站结构对搜索引擎友好

技术 SEO 更偏向工程层面，确保网站在技术上对搜索引擎友好：

### 1. 网站可访问性与状态码

- 重要页面应该返回 **200** 状态码；
- 正确处理 301/302 重定向，避免链式跳转；
- 404 页面存在但不滥用（不存在的 URL 应该返回 404）。

### 2. URL 结构与站点结构

- URL 简洁、语义化，例如：
  - `/seo/intro-1/` 比 `/post?id=1234` 更有利于理解；
- 分类和目录清晰，避免过深的嵌套（比如 6 层路径）。

对于静态博客（比如你现在看的这个 Jekyll 博客），好的 URL 结构可以直接在配置里用 `permalink` 控制。

### 3. Sitemap 与 robots.txt

- `sitemap.xml` 帮助搜索引擎发现你站点的所有重要 URL；
- `robots.txt` 告诉搜索引擎哪些路径可以抓取、哪些应该避免。

### 4. 页面性能与移动端体验

- 页面加载速度太慢，会显著影响用户体验，也会作为 SEO 信号之一；
- 移动端适配（响应式设计）已经是基础要求。

### 5. 结构化数据（Schema.org）

- 对某些类型的页面（文章、FAQ、产品等），可以添加结构化数据标记；
- 有助于搜索引擎在搜索结果中展示「富结果（Rich Snippets）」。

## 四、如何在一个技术博客上实践这些 SEO 做法？

以你现在这个技术博客为例（使用 Jekyll + GitHub Pages）：

- 站内 SEO：
  - 每篇文章有清晰的标题、摘要与合理的关键词；
  - 系列文章之间互相链接（比如这篇链接 1/3 和 3/3）。
- 站外 SEO：
  - 把优质文章分享到开发者社区，放在 GitHub 项目 README 中；
  - 不追求短期外链数量，而是长期积累自然引用。
- 技术 SEO：
  - 保持 URL 和目录结构清晰；
  - 提供 `sitemap.xml` 和 `robots.txt`；
  - 通过轻量主题和静态构建保持较好的性能。

## 五、小结：SEO 技术是为「好内容」服务的

这一篇《SEO 入门必看 2/3》主要讲了：

- 站内 SEO：标题、关键词、内链、内容结构；
- 站外 SEO：外链、品牌信号、社交影响；
- 技术 SEO：网站结构、可访问性、性能、结构化数据。

记住一点：

> 现代 SEO 技术的意义不是「骗过搜索引擎」，而是 **帮助搜索引擎更好地理解和分发你的好内容**。
